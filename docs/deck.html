The one hard rule

It can't cost us money
when a new user shows up.

$0 COGS. No cloud LLM bill per lesson, no per-user server. So the model has to run on the user's own device — and that single rule drives every decision in the product.

The insight

The tutor's failures are about discipline, not IQ.

The scenarios Maestro slips on aren't knowledge tests — "don't leak the answer," "don't rubber-stamp wrong work," "catch the discouraged student." A small model fails these for lack of discipline, and discipline we can engineer. The $0 constraint becomes the strength.

The differentiator

A discipline layer around a small model.

  • Always-on prompt discipline — explain don't dump, signal every shift, show before you test.
  • Per-turn skills — empathy on distress, a nudge in a challenge, never rubber-stamp wrong work, exact math.
  • A $0 verifier — checks the model's own arithmetic and blocks answer-leaks, in code.
  • A mastery quiz — objective, no self-marking; the lesson only completes when it's passed.

Measured with a reproducible in-browser eval — the engine lifts the model on the exact scenarios that matter.
